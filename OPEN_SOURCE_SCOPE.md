# 球悟AI分层开放说明

## 开放目的

球悟AI响应企业赛题对项目开源和技术共享的倡议，公开产品外围工程、RK3588端侧集成框架和标准化接口，使评审方能够审阅系统架构、工程组织、设备接入、任务运行、结果展示与端云协同链路。

本项目同时面向后续真实产品部署。自研模型、数据体系、关键算法和生产参数具有持续研发与商业交付价值，因此采用“产品框架开源、核心运行时受控”的分层方式。该方式公开完整系统边界和协同关系，但不公开足以直接复制核心AI能力的实现材料。

## 已开放内容

- `qiuwu.py`统一产品入口与双模式任务组织；
- PyQt触控界面、模式切换、任务状态、视频回放、比分与事件展示；
- OV13855/V4L2通用预览、录制和摄像头设备接口；
- 子进程管理、日志转发、结果文件监视和标准化解析；
- 比赛视角与个人训练两条主体Pipeline及完整阶段调用关系；
- 四类模型组件接口、跨模块数据结构和任务结果契约；
- RK3588三核NPU Worker运行计划及公开校验逻辑；
- 公开产品层与私有AI运行时之间的命令契约；
- 比赛模式、个人训练模式及任务结果的稳定数据契约；
- Unix Domain Socket、MQTT通知和OSS文件归档的通用适配；
- 不含真实凭据的配置模板、技术文档和演示材料。

上述内容可以独立用于理解和复用嵌入式产品的软件框架，也可通过兼容`runtime_bridge.py`与`contracts/`的第三方实现替换AI运行时。

## 已公开但不披露实现细节的能力

- 比赛视角TrackNet与侧视TrackNet用于高速网球时序热图追踪；
- 球场关键点模型用于场地语义定位和标准球场坐标映射；
- MediaPipe Pose与MS-TCN++用于人体骨架感知和连续动作时序分析；
- 多源事件理解与规则引擎用于落地、场区、回合和比分结果组织；
- RK3588三核NPU、CPU和多媒体链路组成异构端侧运行架构；
- 端侧可离线形成训练或判罚闭环，并可选择将结果接入云端。

这些内容用于说明系统原理、应用和模块关系，不包含可直接复现核心能力的网络参数、数据、代码路径或业务规则。

## 完整系统如何验证

公开仓库提供架构、接口和外围工程；完整AI运行时在实际设备部署包中提供。评审方可结合以下材料验证项目完成度：

1. 公开源码：审阅触控界面、摄像头、端侧任务、结果解析和云适配；
2. 技术文档：审阅模型原理、RK3588部署链路、性能数据和系统设计；
3. 演示视频：查看个人训练、双人对打、端侧处理和前后端服务效果；
4. 实机演示：验证私有运行时与公开产品层通过稳定契约协同运行；
5. 标准化结果：核对视频、轨迹、事件、动作和判罚等输出文件。

## 许可与贡献

公开程序源代码采用`AGPL-3.0-only`。模型、数据、私有运行时、品牌、技术文档、演示素材和硬件设计不因源代码开源而自动获得授权。

外部贡献应限于公开模块。请勿在Issue、Pull Request、日志或附件中提交模型、数据、密钥、内部参数、私有运行时源码或未公开设备资料。
